Privacy Policy

Last updated: 2026-05-27

1. Scope

This policy applies to the personal automation software (the "App") that connects to Google Calendar on behalf of a single authenticated user. The App is operated privately by the developer for personal use, and is not offered as a public service.

2. Information We Access

With the user's explicit OAuth consent, the App accesses:

  • Google Calendar events on the user's calendars (read and write) via the scopes https://www.googleapis.com/auth/calendar and https://www.googleapis.com/auth/calendar.events.
  • The list of calendars the authenticated account owns or is subscribed to.

The App does not request access to email, contacts, Drive files, or any other Google services.

3. How We Use the Information

Calendar data is read and written solely to:

  • Create, read, update, and delete events that represent the user's tasks, reviews, and reminders.
  • Synchronize the user's personal task system with their calendar.

4. Where Data Is Stored

OAuth tokens and any cached calendar data are stored on the user's own personal NAS (Network Attached Storage) device, in restricted-permission directories (chmod 600) accessible only to the user. No data is transmitted to any third-party server other than Google's own APIs.

5. Sharing

The App does not share, sell, or transmit any user data to third parties. All data processing happens locally on the user's NAS.

6. Retention & Deletion

The user may revoke the App's access at any time at Google Account permissions. The user may delete locally cached tokens and data at any time by removing the relevant files from the NAS.

7. Security

OAuth credentials and tokens are stored with restricted file permissions (700/600) and are accessible only to the OS user running the automation. The NAS is reachable only via the user's private Tailscale network.
